## 参考站点观察

### 参考站点 1：视觉族

参考链接：
- https://www.shijuezu.com/

UI 观察：
- 首页首屏非常直接，进入页面后立刻给用户一个明确动作入口
- 导航很轻，但功能入口明确，属于“功能优先型首页”
- 页面结构偏“先让你理解能做什么，再逐层解释价值”

适合借鉴的点：
- 首屏不绕
- 入口清楚
- 说明区分层清晰

不适合直接照搬的点：
- 更像工具站，不是品牌型产品官网
- 产品气质和桌面软件成熟感较弱

### 参考站点 2：Cherry Studio

参考链接：
- https://www.cherry-ai.com/
- https://docs.cherry-ai.com/

UI 观察：
- 公开信息结构更偏“产品官网 + 功能矩阵 + 下载入口”
- 复杂能力通过模块化方式表达
- 下载入口、文档入口与能力介绍彼此联动

适合借鉴的点：
- 复杂产品能力的分层表达
- 产品能力与使用场景同时出现
- 官网和文档体系天然联动

不适合直接照搬的点：
- 产品范围更广
- 如果首页塞太多能力点，首版会显得过重

### 观察结论

适合 xosite 的方向不是完全偏向其中一个，而是做融合：
- 借鉴视觉族的“首屏直接、入口清楚”
- 借鉴 Cherry Studio 的“产品官网式表达、模块化功能说明、下载与文档联动”

更适合 xosite 的首版官网形态应是：
- 一个有品牌感的桌面软件官网
- 一个可信、清晰、成熟的财务工作台官网
- 不是纯工具站
- 也不是过重的 AI 营销站

## xosite 官网优先方案

### 当前采用方案

当前优先采用：
- `XplorOne 官网方案 v3`

采用原因：
- 比泛 AI 官网路线更贴合 XplorOne 当前真实状态
- 更符合你现有软件的明亮、可信、轻专业气质
- 能更好地把“本地优先财务工作台”讲清楚，而不是把 AI 讲成噱头

### 建站目标定位

首版网站建议定位为：
- `桌面软件产品官网`
- 辅助承载独立开发者项目身份，但不做纯个人名片站

首版统一主定位文案：
- `为超级个体与自由职业者打造的本地优先财务工作台`

核心目标：
- 先把 xosite 讲清楚
- 再把核心能力展示清楚
- 最后把下载、更新和后续接入入口建立起来

### 官网主叙事

全站统一围绕一句话展开：
- `XplorOne 不是又一个记账 App，而是一套帮助超级个体和自由职业者看懂资金状态、经营节奏与真实收支的本地财务工作台。`

建议长期反复使用的 4 个子句：
- 不必先学会计，才能把财务理清
- 你看到的不只是流水，而是结构
- 数据优先留在自己电脑里
- AI 帮你查、帮你看、帮你联动，而不是只会聊天

### 目标用户

第一层核心用户：
- 超级个体
- 自由职业者

第二层用户：
- 小型工作室主理人
- 轻经营者

第三层用户：
- 对隐私敏感、偏爱本地工具的用户

第四层补充用户：
- 技术用户
- AI 工具用户

表达原则：
- 首页前半段优先服务第一层核心用户
- MCP、本地 API、模型兼容等技术表达放在首页后半段或二级页面

### 首版信息架构

首页建议采用 8 个核心区块：

1. Hero 首屏
- 产品名
- 一句话定位
- 真实产品截图主视觉
- 主按钮：
  - 下载 Windows 版
  - 查看功能演示
- 首屏截图建议优先从这些页面中选择：
  - 首页 Dashboard
  - 流水页
  - 报表页

2. 用户问题区
- 先讲用户真正关心的问题，不急着先讲功能
- 建议问题卡方向：
  - 这个月到底是赚了还是只是忙了
  - 手上的钱安全吗，还能撑多久
  - 哪些支出正在慢慢拖累自己
  - 账户、预算和分类是不是已经乱了

3. 对比区
- 对比对象：
  - 普通记账 App
  - 传统财务软件
  - XplorOne
- 对比维度建议：
  - 核心目标
  - 用户门槛
  - 数据方式
  - 财务视角
  - AI 角色
  - 适合人群

4. 核心能力区
- 按真实产品能力拆成 5 张能力卡
- 推荐模块：
  - 多账本与账户结构
  - 分类、预算与秩序建立
  - 专业图表与经营视角
  - AI 查询与分析
  - 跨页面联动

5. 产品截图区
- 首发建议放 5 张：
  - 首页 Dashboard
  - 日历页
  - 流水页
  - 预算页
  - 报表页
- 展示方式优先：
  - 横向截图卡片
  - 每张图配一句话

6. 本地优先与信任区
- 先用人话表达：
  - 数据默认保存在你的电脑里
  - 不是先上云，再谈隐私
  - AI 能力有边界，不是随意改数据
  - 查询、展示和后续操作尽量统一口径
- 再补一个“技术细节”折叠区：
  - Electron
  - React + TypeScript
  - 本地数据库
  - 统一 service 层
  - Local API / MCP

7. 使用场景区
- 超级个体
- 自由职业者
- 小型工作室主理人
- 隐私敏感用户

8. FAQ + 收口 CTA
- FAQ 首版建议覆盖：
  - XplorOne 适合谁
  - 和普通记账软件有什么不同
  - 数据真的只在本地吗
  - AI 会不会把数据上传出去
  - 现在支持哪些平台
  - 需要 API Key 吗
  - 后续会收费吗
  - 怎么备份和迁移
- 收口 CTA 建议：
  - 下载体验
  - 查看更新日志

### 视觉方向建议

首版建议视觉关键词：
- 清晰
- 可信
- 明亮
- 轻专业
- 有秩序
- 友好
- 带一点生活感

建议避免：
- 整站深黑
- 大面积玻璃拟态
- 霓虹发光
- 很重的 AI SaaS 模板风
- 官网比产品更炫

建议方向：
- light-first 方案
- 浅灰白、略偏冷调背景
- 白色或极浅灰白卡片
- 轻边框、轻阴影
- 主文字使用深蓝灰而不是纯黑
- 主 CTA 使用产品里的橙色
- 品牌强调区沿用产品中的蓝色语言
- 图表辅助色继续使用青绿、绿色、蓝紫、橙红
- 官网像“产品 UI 的品牌展示版”，而不是另一套风格体系

### 与当前软件能力的映射建议

基于当前项目资料，网站内容不应只停留在“我做了个记账软件”，而应真实映射现有体系：

- 产品层：
  - 多账本
  - 预算并库
  - 数据备份/恢复/迁移
  - 个性化设置
  - 专业图表

- 技术层：
  - Electron 桌面端
  - 本地优先
  - SQLite
  - Local API
  - Read-only MCP
  - GitHub + Vercel 官网发布链路

- AI 层：
  - 查询
  - 录入
  - 分析
  - 自由对话

这意味着网站首版可以按照“三层表达”来写：
- 产品能力
- 技术架构
- AI 增强能力

同时要注意表达顺序：
- 首页前半段优先讲用户问题、财务结构与本地优先
- 技术层与 AI 层不消失，但不抢首页第一叙事位
