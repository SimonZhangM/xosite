### 起步记录

新增：
- 建立 `WEBSITE_BUILD_PROGRESS.md`，用于持续追踪建站计划与阶段进度
- 建立 `WEBSITE_CHANGELOG.md`，用于记录后续网站建设过程中的重要变更
- 在进度文档中补充参考站点分析与 `xosite` 首版网站初步建构计划
- 将朋友提供的 `XplorOne 官网方案 v3` 吸收到进度文档，作为当前优先官网方案

当前状态：
- 已完成首版网站代码初始化与页面实现
- 已确认后续采用 GitHub 连接 Vercel 的自动部署工作流
- 已形成首版官网方向：以“桌面软件产品官网”为主，而不是纯个人名片页
- 首页叙事、视觉方向与目标用户定义，现以 v3 方案为主

### 首版网站落地

新增：
- 初始化 `Next.js + TypeScript` 官网工程，并落地 4 个页面：`/`、`/download`、`/faq`、`/changelog`
- 完成首页 8 个区块的正式实现，使用真实产品截图作为主视觉与界面预览素材
- 新增统一导航、Footer、状态标签体系与响应式布局
- 新增基础 SEO 能力：`metadata`、Open Graph、`sitemap.xml`、`robots.txt`
- 新增下载页真实占位策略：安装包暂未开放，但保留发布进度、GitHub 状态与更新日志去处

调整：
- 首页主定位改为 `超级个体与自由职业者的本地优先财务工作台`
- 首页次按钮由“查看功能演示”调整为“查看界面预览”
- 本地优先与 AI 边界文案统一改为分层表达，不做绝对化承诺
- `changelog` 页面采用“官网更新 / 产品更新”双时间线展示

验证：
- `npm run lint` 通过
- `npm run build` 通过
- 本地开发服务访问 `/`、`/download`、`/faq`、`/changelog` 均返回 `200`
- 本地生产服务访问 `/`、`/download`、`/faq`、`/changelog` 均返回 `200`
